# 📖 نظام إدارة مركز القرآن الكريم (Quran Center Management System)

تطبيق متكامل وحديث لإدارة مراكز تلاوة القرآن الكريم وتحفيظه، مصمم لمعمارية حقيقية وسريعة:
**Next.js (App Router) + Node.js + Prisma ORM + PostgreSQL**.

---

## 🌟 الميزات الأساسية:
1. **شاشة التسميع اليومي للمحفظ (Mobile First)**:
   - واجهة لمسية سريعة وسهلة الاستخدام للشيوخ على الهواتف الذكية.
   - تسجيل الحضور (حاضر / غائب / عذر / حضر ولم يسمّع).
   - تسجيل الحفظ الجديد، المراجعة، والسرد لكل طالب في الجلسة نفسها.
   - منع الحفظ في تواريخ تخرج عن أيام جدول الحلقة مع تنبيهات ملونة.

2. **لوحة تحكم مدير المركز (Manager Dashboard)**:
   - **المتابعة اليومية**: إحصائيات فورية وتنبيه بارز للحلقات التي لم تسجّل اليوم.
   - **التنبيهات الإدارية التشغيلية**: كشف الحلقات دون تسجيل، والحلقات دون محفظ، والمسودات القديمة، والحسابات المعطلة.
   - **متابعة الطلاب المتأخرين**: تصفية الطلاب حسب الغياب المتكرر وعدم الإنجاز، مع زر استخراج تقرير ولي الأمر مباشرة.
   - **إدارة الحلقات والطلاب والمستخدمين**: نقل الطلاب، تفعيل/تعطيل الحسابات بـ Confirm Dialogs آمنة.
   - **سجل التدقيق (Audit Log)**: كشف وقائعي لجميع التغييرات الإدارية الحساسة.

3. **تقرير متابعة ولي الأمر (Parent Report)**:
   - تقرير فخم منسق ومجهز للطباعة المباشرة بصيغة A4.
   - تقييم آلي منطقي (ممتاز، جيد جداً، جيد، يحتاج متابعة) بناءً على نسبة الحضور والإنجاز بالصفحات.

4. **التقارير الشهرية والتصدير الثلاثي**:
   - تصدير كشوفات الإنجاز والاختبارات بصيغ Excel XML، و **CSV UTF-8 BOM** الداعم للعربية في Microsoft Excel، و **PDF**.

---

## 🚀 متطلبات التشغيل (Prerequisites):
- **Node.js**: الإصدار 18.18.0 أو 20.x أو أحدث.
- **PostgreSQL**: الإصدار 14 أو أحدث.
- **npm**: الإصدار 9 أو أحدث.

---

## ⚙️ طريقة الإعداد والتشغيل المحلي (Local Setup):

### 1. تثبيت الحزم (Install Dependencies):
```bash
npm install
```

### 2. إعداد متغيرات البيئة (Environment Variables):
قم بنسخ ملف `.env.example` إلى `.env` وتحديث بيانات الاتصال:
```bash
cp .env.example .env
```
محتوى `.env`:
```env
DATABASE_URL="postgresql://postgres:password@localhost:5432/quran_center_db?schema=public"
SESSION_SECRET="your-32-character-long-secret-key-change-this-in-production"
NODE_ENV="development"
```

### 3. إنشاء وترحيل قاعدة البيانات (Database Migration & Generate):
```bash
npm run db:migrate
```

### 4. بذر البيانات الأولية وإنشاء أول مدير (Seed Initial Data):
```bash
npm run db:seed
```
*عند تشغيل هذا الأمر، يتم تلقائياً إنشاء الحسابات الأولية وحلقة تجريبية وطلاب:*
- **مدير المركز**: اسم المستخدم `admin` | كلمة المرور `Manager@123456`
- **الشيخ المحفّظ**: اسم المستخدم `teacher1` | كلمة المرور `Teacher@123456`
- **المختبر**: اسم المستخدم `examiner1` | كلمة المرور `Examiner@123456`

### 5. تشغيل خادم التطوير (Development Mode):
```bash
npm run dev
```
افتح المتصفح على `http://localhost:3000/login`.

## 🏢 طريقة النشر على السيرفر (Production Deployment):

1. **ضبط المتغيرات في السيرفر**:
   أكد تفعيل `NODE_ENV=production` ومفتاح `SESSION_SECRET` عشوائي قوي (32+ حرف).

2. **تنفيذ الترحيل الإنتاجي**:
   ```bash
   npm run db:migrate:deploy
   ```

3. **بناء النسخة الإنتاجية (Build)**:
   ```bash
   npm run build
   ```

4. **تشغيل الخادم (Start)**:
   ```bash
   npm run start
   ```

## 📋 دليل التجربة الميدانية داخل المركز (Field Pilot Checklist):

1. **المدير**:
   - تسجيل الدخول بـ `admin` / `Manager@123456`.
   - تغيير كلمة المرور أو إنشاء حساب مدير مخصص.
   - إنشاء الحلقات، تعيين الشيوخ، وإضافة الطلاب.
2. **الشيخ**:
   - تسجيل الدخول بـ `teacher1` / `Teacher@123456` من الهاتف.
   - تسجيل تسميع اليوم (حاضر / لم يسمع / إنجاز صفحات).
   - اعتماد الجلسة واستخراج تقرير ولي الأمر لأحد الطلاب.
3. **طباعة التقرير والتصدير**:
   - تجربة زر **"طباعة / حفظ PDF"** لتقرير ولي الأمر.
   - تجربة تصدير **CSV UTF-8** وفتحه في Microsoft Excel للتأكد من اللغة العربية.

- قم بتغيير كلمة مرور الحساب الرئيسي (`admin`) فور الدخول لأول مرة.
