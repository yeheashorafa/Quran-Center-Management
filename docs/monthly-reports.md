# التقارير الشهرية

أضيفت تقارير شهرية موحدة للشيخ ومدير المركز والمختبر.

## الصلاحيات

- `reports.export.own`: يسمح للشيخ بتصدير بيانات الحلقات التي كان معيناً عليها خلال الشهر المحدد فقط.
- `reports.export.all`: يسمح للمدير والمختبر بالتصدير على مستوى المركز.

يتم التحقق من الصلاحية والنطاق مرة أخرى داخل الخادم، ولا يعتمد النظام على الخيارات المرسلة من الواجهة فقط.

## التقارير المتاحة

### الشيخ

تقرير شهري شامل يتضمن:

- أيام الجلسات المتوقعة والمسجلة.
- الحضور والغياب والعذر و«لم يسمع».
- صفحات الحفظ والمراجعة والسرد.
- إحصائيات كل طالب.
- الاختبارات الرسمية لطلاب الحلقة.

### مدير المركز

- تقرير شامل لكل المركز أو لمرحلة أو حلقة محددة.
- تقرير مستقل للاختبارات الرسمية.
- إمكانية تضمين الاختبارات الملغاة.

### المختبر

- تقرير الاختبارات الرسمية حسب الشهر والمرحلة والحلقة.
- إمكانية تضمين الاختبارات الملغاة.

## صيغ التصدير

### Excel

يتم إنشاء ملف Spreadsheet XML متعدد الأوراق ومتوافق مع Microsoft Excel، ويحتوي على أوراق للملخص والحلقات والطلاب والاختبارات وفق نوع التقرير.

### PDF

يتم إنشاء PDF حقيقي من قالب HTML عربي بواسطة Chrome أو Chromium مثبت على الخادم. يبحث النظام تلقائياً عن المتصفحات الشائعة في Windows وmacOS وLinux.

عند عدم اكتشاف المتصفح يمكن تحديد مساره في `.env`:

```env
REPORT_BROWSER_EXECUTABLE_PATH="C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe"
```

أو في Linux:

```env
REPORT_BROWSER_EXECUTABLE_PATH="/usr/bin/chromium"
```

## API

```text
GET /api/reports/monthly
```

المعاملات:

```text
month=YYYY-MM
kind=COMPREHENSIVE|EXAMS
format=excel|pdf
stageId=UUID (اختياري)
halaqaId=UUID (اختياري)
includeVoided=true|false
```

كل عملية تصدير ناجحة تسجل في `audit_logs` باسم:

```text
MONTHLY_REPORT_EXPORTED
```
