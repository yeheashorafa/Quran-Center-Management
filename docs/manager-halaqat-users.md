# إدارة المستخدمين والحلقات

## إدارة المستخدمين

مدير المركز الذي يملك الصلاحية `users.manage` يستطيع:

- إنشاء مستخدم بدور الشيخ أو المدير أو المختبر.
- تعيين اسم مستخدم واسم ظاهر وكلمة دخول.
- إيقاف المستخدم دون حذف حسابه أو سجلاته.
- إعادة تفعيل المستخدم.

كلمة الدخول تُشفّر باستخدام Argon2id قبل الحفظ. عند إيقاف المستخدم تُلغى جلساته النشطة فوراً. لا يستطيع المدير إيقاف الحساب الذي يستخدمه حالياً.

## إدارة الحلقات

مدير المركز الذي يملك الصلاحية `halaqat.manage` يستطيع:

- إنشاء حلقة داخل البرنامج الأساسي.
- اختيار المرحلة.
- اختيار الشيخ المسؤول.
- تحديد أيام الحلقة.
- تحديد تاريخ بدء الجدول.
- إيقاف الحلقة دون حذف الطلاب أو الجلسات أو التعيينات التاريخية.
- إعادة تفعيل الحلقة لاحقاً.

عند إنشاء الحلقة تُنفذ الخطوات داخل Transaction واحدة:

1. إنشاء الحلقة.
2. إنشاء أيام الدوام.
3. إنشاء تعيين الشيخ الأساسي.
4. تسجيل العملية في `audit_logs`.

## مسارات API

```text
POST  /api/manager/users
PATCH /api/manager/users/:userId/status
POST  /api/manager/halaqat
PATCH /api/manager/halaqat/:halaqaId/status
```

كل مسار يتحقق من:

- الجلسة من PostgreSQL.
- الصلاحية المطلوبة.
- Same-Origin للطلبات المعدّلة للبيانات.
- صحة المدخلات بواسطة Zod.
