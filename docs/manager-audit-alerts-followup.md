# سجل التدقيق والتنبيهات ومتابعة الطلاب

## سجل التدقيق الإداري

المسار:

```text
GET /api/manager/audit-logs
```

الصلاحية المطلوبة:

```text
audit.read
```

يدعم البحث والتصفية حسب:

- المستخدم المنفذ.
- العملية.
- نوع الكيان.
- الفترة الزمنية.
- البحث باسم المستخدم أو العملية أو الكيان.
- التصفح على صفحات لحماية الأداء عند نمو السجل.

يعرض السجل القيم القديمة والجديدة والبيانات الإضافية وRequest ID وعنوان IP عند توفرها. السجل للقراءة فقط ولا توجد عملية حذف من الواجهة.

## التنبيهات التشغيلية

المسار:

```text
GET /api/manager/alerts?date=YYYY-MM-DD&lookbackDays=7
```

التنبيهات ديناميكية وتُحسب من البيانات الحالية، ولا تُخزن في جدول منفصل. تشمل:

- حلقة مجدولة لم تبدأ تسجيل الجلسة.
- جلسة اليوم ما زالت مسودة أو ناقصة.
- مسودات قديمة لم تُعتمد.
- حلقة نشطة بلا شيخ أساسي سارٍ.
- حلقة مرتبطة بشيخ حسابه غير نشط.
- حلقة متوقفة وما زالت تحتوي تسجيلات طلاب نشطة.
- ملخص الطلاب الذين يحتاجون متابعة.

بمجرد معالجة السبب يختفي التنبيه في التحديث التالي.

## الطلاب الذين يحتاجون متابعة

المسار:

```text
GET /api/manager/follow-up
```

المعاملات:

```text
from=YYYY-MM-DD
to=YYYY-MM-DD
stageId=UUID       اختياري
halaqaId=UUID      اختياري
attendanceThreshold=70
```

قواعد المتابعة الحالية:

- ثلاث حالات غياب متتالية أو أكثر.
- ثلاث حالات غياب ضمن الفترة أو أكثر.
- نسبة حضور أقل من الحد المختار مع وجود ثلاث جلسات مسجلة على الأقل.
- حالتا «لم يسمع» أو أكثر.
- ثلاث جلسات حضور أو أكثر دون صفحات إنجاز.
- تكرار جلسات الحضور ذات الصفحات الصفرية.

يتم ترتيب الطلاب حسب أولوية السبب، ثم نسبة الحضور. يبقى القرار الإداري النهائي للمدير، لأن المؤشرات تساعد في المتابعة ولا تمثل حكماً تلقائياً على الطالب.

memorization_sessions
